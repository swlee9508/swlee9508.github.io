AES(대칭키) 방식의 암/복호화

AES(대칭키) 방식의 암/복호화 란??

AES 암/복호화 방식은 벨기에의 암호학자 Joan Daemen(존 대먼) 과 Vincent Rijmen(빈센트 라이먼) 에 의해서
개발된 암/복호화 방식으로 대표적인 대칭키 암/복호화라고 불리면서
수신자와 송신자가 동일한 키를 가지고 암/복호화 하는 방식을 이야기 합니다.

AES 암호화는 아래와 같은 절차로 이루워집니다.

  • 메세지를 보내는 수신자와 송신자는 동일한 비밀키를 공유합니다.
  • 수신자는 메세지를 서로 공유한 비밀키로 암호화 한 후에 전송 합니다.
  • 송신자는 전달 받은 암호문을 비밀키로 복호화 합니다.

    • 이 방식을 이용하면 얻을 수 있는 장/단점 이 존재합니다.

    장점

  • 암/복호화의 속도가 빠릅니다.
  • 이미 사전의 키를 공유하기 때문에 암호문 사이즈의 증가가 없습니다.

    • 단점

  • 사전에 키 공유를 어떻게 주고 받을 방법의 대한 문제를 해결 해야 됩니다.

    • AES 암복호화는 이론적인 부분에서는 많은 부분에서 설명이 필요하지만
    이번 포스팅에서는 그 전 포스팅에서 설치한 Pycrptodome 에 포함된 AES 라이브러리를 살펴보면서 간단하게 설명하고 코드를 분석하겠습니다.

    Ramdome은 각 size에 맞춘 key 와 IV 값을 생성 시켜 주기 위해서 AES는 구현된 AES 함수를 사용하기 위해서 import 시켜주었습니다.

    AES 암호화는 블록 단위로 암호화가 진행 되는데 각각의 블록의 대한 사이즈를 정의 해주어야 됩니다.
    보통 AES 암호화 작업에서 많이 사용되는 블록 크기는 16 byte 이기 떄문에 16 byte 의 사이즈를 가진 블록을 생성 해주면 됩니다. 추후 IV 값을 설정 하는데에 이용 됩니다.
    그리고 공유할 비밀 키의 사이즈를 정해주어야 됩니다. 16 byte 블록 사이즈 기준의 키 사이즈는 32 byte 값으로 설정해 줍니다.

    그 다음 우리는 LETTERS 라는 변수 명을 가진 변수에 대소문자의 전체 길이를 저장 하겠습니다.

    다음으로 암호화 할 메세지를 String 타입으로 작성 하여 줍니다.
    이번 포스트에서는 "hello CaesarCipher" 라는 메세지를 암/복호화 하겠습니다.

    메세지를 저장할 변수를 plainT 로 정의 하고 값을 저장 합니다.
    plainT = "hello CaesarCipher"

    평문 생성이 끝나면 우리는 암호화시 이용할 key 값을 생성해야 됩니다.
    key 값을 생성 하기 위해 random 패키지를 import 시켜주겠습니다.

    그 다음 key 라는 변수에 0~문자열 총 길이 사이에 랜덤인 숫자값을 하나 정하여서 저장 하겠습니다.

    key = random.randrange(0,len(LETTERS))
    randrange 는 랜덤을 돌릴 범위를 이야기 하고 len(LETTERS)는 LETTERS 의 전체 길이를 의미 합니다.

    키 값과 평문이 생성이 됬다면 암/복호화 메서드를 만들어 주겠습니다.

    먼저 암호화문을 생성하는 메서드 코드 입니다.

    일일이 암호화 방식을 적용 할 수 있지만 위처럼 메서드로 생성을 하면 호출하여 언제든지 재사용 할 수 있어서 메서드 방식으로 코드를 생성하였습니다.

    1. def encrypt()안에 message, key, LETTERS 는 각각 (평문, 키, 시져를 돌릴 문자열의 총 수)를 의미 합니다.

    2. encrypt_message='' 는 새로운 문자열 변수를 선언 합니다. 이 안에는 암호문이 들어갈 변수 입니다.

    3. 시저 암호화 방식에서 가장 중요한 것은 각 평문 한 글자 마다. 특정 키 값 만큼 자리를 이동하여 만들어 주는 암호 방식입니다.

    그 방식을 이번 코드에서는 반복문(for)문을 이용한 방식을 이용하여 구현을 하였습니다.

  • 받아온 평문의 문자를 한 글자씩 분할하여 symbol 이라는 변수에 저장 합니다.
  • 만약 그것이 영문 대 소문자가 저장 되어 있는 LETTERS안에 존재하면 그 symbol 값을 LETTERS의배열의 위치에서 key 값을 더해 준 다음에 전체 LETTER 길이 만큼의 나머지 연산을 진행 합니다.
  • 그리고 계산된 값을 num 변수에 저장하여 그 숫자를 가지고 LETTERS 배열에서 그 자리에 있는 영문 글자를 찾아서 encrypt 변수에 하나씩 붙여 줍니다.
  • 이와 같이 평문이 모두 끝날때 까지 반목문을 실행 하다 보면 암호문이 완성이 됩니다.
  • else 부분은 만약에 평문에 특수 문자나 LETTER 배열에 존재하지 않은 값이 있으면 그것은 그냥 암호문 변수에 그대로 붙여줍니다.

    • 생성된 암호문을 푸는 복호화 메서드 코드 입니다.

    전반적인 부분은 암호화 과정하고 동일합니다. 하지만 다른 부분 하나는 기존의 키 값을 더해서 자리 이동을 시켜주었다면 이번에는 반대로 키 값을 다시 뺴내 주면 원래 단어가 선택이 되고 그것을 다시 decrypt 변수의 저장 하면 원래 평문이 다시 출력 됩니다.

    최종적으로 완성 된 CasarCipher 암/복호화 방식의 전체 코드 입니다.

    메인 문장 안에 평문을 입력하고 랜덤으로 키 값을 생성한 후 기존에 만들었던 암/복호화 메서드를 이용하면 아래의 사진과 같은 결과 값을 확인 할 수 있습니다.
